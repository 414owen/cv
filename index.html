Regex engine

Implemented our in-house regex engine from scratch. This supports non-mainstream features, such as complement, and intersection, and compiles regex exponentially faster than the rust regex crate.

As an example, the regex crate compiles (a|b)*a(a|b){14} to a minimal DFA in 43s, and the library I developed compiles it in 26ms, so 1653x faster.

This regex engine is a key reason we're competitive in terms of Time To First Token.

Pangram - A universal grammar converter

Implemented our universal grammar conversion tool, which is a micropass transpiler, supporting many middle-end optimization and transformation passes.

It has frontends for:

  • Lark
  • GBNF
  • JSON Schemas
  • Tree Sitter
  • XGrammar structural tags

Backends for:

  • Lex/YACC
  • Lark

As well as these optimization/transformation passes:

  • Inlining
  • Simplification
  • Rule Deduplication
  • Tree Pruning
  • EBNF → BNF
  • Grammar → regex
  • Wordlevel → bytelevel

JSON Schema support

To support JSON Schemas, we convert said schemas to grammars. This is a non-trivial task (in fact, it's an ongoing area of academic research), because modern JSON schemas are sets of constraints, which all interact with each other in different, and sometimes surprising, ways. Solving this means doing a lot of constraint propagation. I ended up writing two separate IRs to represent JSON Schemas specifically, as well as four transformation passes, before spitting out an EBNF-IR (the middle-end IR of Pangram, above).

I spearheaded this work, leading a small but extremely capable team of compiler engineers.
